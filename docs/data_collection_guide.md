# 📋 생기부 데이터 수집 가이드

## 개요

입시 컨설턴트와 협업하여 생기부 PDF를 JSON 형식으로 변환할 때 참고하는 가이드입니다.

---

## 📁 파일 위치

| 파일 | 경로 | 설명 |
|------|------|------|
| **스키마** | `data/schemas/student_record_schema.json` | JSON 데이터 형식 정의 |
| **예시** | `data/examples/student_record_example.json` | 실제 작성 예시 |
| **수집 데이터** | `data/raw/students/` | 변환된 JSON 파일 저장 위치 |

---

## 🔐 비식별화 규칙

### 필수 비식별화 항목

| 원본 정보 | 비식별화 방법 |
|-----------|---------------|
| 학생 이름 | `STU_연도_일련번호` (예: STU_2024_001) |
| 학교명 | 삭제 |
| 대학명 | 티어로 변환 (최상위권/상위권/중상위권/중위권) |
| 지역 | 서울권/수도권/비수도권 |
| 내신 등급 | 등급대로 변환 (1등급대, 2등급대 등) |
| 수상 실적 | 대회명 일반화 (예: "XX대회 대상" → "전국 대회 입상") |

### 대학 티어 기준 (예시)

| 티어 | 설명 |
|------|------|
| 최상위권 | SKY, 의대, 최상위 학과 |
| 상위권 | 주요 10개 대학, 상위 학과 |
| 중상위권 | 인서울 주요 대학 |
| 중위권 | 인서울/지거국 |

---

## ✅ 수집 대상 학생 기준

**핵심 조건: 성적 대비 상향 진학 (is_overperform: true)**

예시:
- 3등급대 → 상위권 대학 합격
- 4등급대 → 중상위권 대학 합격
- 비교과/세특이 뛰어난 학종 합격 사례

---

## 📝 컨설턴트 작성 항목

### 각 활동별 평가

| 항목 | 설명 | 점수 |
|------|------|------|
| `uniqueness_score` | 활동의 신박함/차별성 | 1-5 |
| `effectiveness_score` | 합격 기여도 | 1-5 |
| `major_connection` | 전공과의 연결성 설명 | 텍스트 |
| `tags` | 활동 키워드 태그 | 배열 |

### 종합 분석 (consultant_notes)

| 항목 | 설명 |
|------|------|
| `key_strengths` | 이 학생의 핵심 강점 (3-5개) |
| `winning_activities` | 합격에 결정적이었던 활동 |
| `narrative_summary` | 학생의 스토리/서사 요약 (2-3문장) |

## 🚀 작업 순서

1. **PDF 수집**: 합격 사례 학생 생기부 PDF 확보
2. **비식별화**: 개인정보 제거/일반화
3. **핵심 추출**: 세특/비교과 중 의미있는 활동 선별
4. **JSON 작성**: 스키마에 맞게 변환
5. **검수**: 형식 및 내용 검토

## ❓ 자주 묻는 질문

**Q: 모든 활동을 다 기록해야 하나요?**
> A: 아니요. 합격에 기여했다고 판단되는 핵심 활동만 선별하세요. 학생당 5-10개 정도가 적당합니다.

**Q: NEIS 문장은 그대로 복사해도 되나요?**
> A: 네, 단 학교명/이름 등 개인정보는 반드시 삭제하세요.

**Q: uniqueness_score 기준이 뭔가요?**
> A: 일반적인 활동(독서, 발표)은 1-2점, 차별화된 주제/방법은 3-4점, 매우 독창적인 활동은 5점.
